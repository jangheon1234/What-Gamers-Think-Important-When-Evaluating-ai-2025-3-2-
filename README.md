# 게이머들이 게임을 평가 할 떄 민감하게 반응하는 요소

## 1. 개요 
평소 RPG 게임을 즐겨 하고, 한 명의 플레이어로서 게임을 경험하다 보니, 게임 개발에서는 단순히 개발 능력만큼이나 유저와의 소통이 중요하다는 생각을 하게 됐다. 이런 관점에서 이번 프로젝트를 기획하게 됐다.

---

## 2. 현재 진행도 및 예상 진행 방향 
### 1). 현재 진행도
  크롤링을 대부분 완료하였으나 부정적인 리뷰가 중요한데 대부분 긍정적인 게임만 존재해 부정적인 리뷰가 많은 게임을 찾는 중임
  
### 2). 예상 진행 방향 
  | 단계 | 목표 | 주요 내용 및 방법론 |
  | :--- | :--- | :--- |
  | 1. 감성 기반 이분화 | 칭찬 요소와 불만 요소 명확히 분리 | 긍정/부정 리뷰 그룹별 키워드 추출을 통해 '핵심 매력 요소'와 '시급 보완 요소'를 이분화하여 전략 기초 확보. |
  | 2. 민감도 지수 | 유저 분노 요소의 우선순위 결정 | (부정 언급량)이 아닌 **(부정 감성 강도)**를 활용한 **'민감도 지수'**를 산출하여 객관적인 개선 순위를 확정. |
  | 3. 복합 연관성 분석 | 불만을 폭발시키는 근본적인 패턴 규명 | '버그 + 최적화' 등 두 가지 이상의 민감 키워드가 동시에 언급된 리뷰를 분석하여 복합적 문제의 원인을 파악. |
  | 4. 결론 | 리뷰를 크롤링하여 입력시 평가 | 분석 결과를 바탕으로 현재 게임의 문제점을 파악 및 시각. |

## 3. 데이터 확보 및 분석 방법 (상세 계획)

#### 3.1. 데이터 수집 및 보완 계획

| 구분 | 내용 |
| :--- | :--- |
| **출처** | Steam, 메타크리틱 (RPG 장르 및 유저 비평이 많은 게임) |
| **규모** | **최종 약 40,000건** 이상의 이용자 리뷰 크롤링 목표 (부정 리뷰 비율 확보를 위해 추가 수집 진행) |
| **특징** | 출시 후 1개월 이상 경과한 게임의 리뷰를 중점 수집하여, 단순 초기 버그가 아닌 **게임의 본질적 문제**에 대한 데이터를 확보합니다. |

#### 3.2. 핵심 분석 기법 (KoELECTRA 기반)

| 분석 기법 | 상세 내용 | 목적 |
| :--- | :--- | :--- |
| **KoELECTRA 기반 감성 분석** | 한국어 특화 모델을 활용하여 리뷰 문장별 **부정 감성 강도(확률)**를 정확하게 측정합니다. | **민감도 지수** 산출을 위한 핵심 정량 데이터 확보. |
| **키워드 빈도 및 정제** | 단순 명사 추출을 넘어, **개발 관련 전문 용어** 및 **비속어/은어**를 정제하여 분석의 정확도를 높입니다. | 유저들이 사용하는 언어에서 핵심적인 불만 키워드를 식별. |
| **복합 연관성 지수** | **'밸런스'와 '현질'** 등 두 개 이상의 키워드 조합에 대한 **분노 증폭 지수**를 계산합니다. | 단순 문제를 넘어선 **근본적인 유저 불만 패턴**을 규명. |

## 4. 기대 효과 (프로젝트 결과 예측)

비록 현재 분석이 진행 중이지만, 최종적으로 이 프로젝트가 달성할 수 있는 **결론 및 기대 효과**는 다음과 같습니다.

* **Top 5 민감 요소 리스트 도출**: 데이터에 기반하여 **게이머들이 가장 민감하게 반응하는 요소 5가지**의 우선순위와 그에 대한 정량적 지표(민감도 지수)를 제시할 수 있습니다.
* **유저 니즈 맵 완성**: 긍정 리뷰 분석을 통해 **유저 만족도를 높이는 핵심 요소**를, 부정 리뷰 분석을 통해 **실패를 유발하는 핵심 요소**를 분류하여 **실질적인 게임 개발 및 운영 전략의 지침**을 제공합니다.
* **정책 제언**: "미흡한 버그 관리에 인력 $X\%$를 우선 투입해야 한다"와 같이, **데이터를 근거로 한 구체적이고 실현 가능한 개선 방안**을 제시하여 프로젝트의 실용성을 증명할 것입니다.
