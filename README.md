# 게이머들이 게임을 평가할 때 민감하게 반응하는 요소

---

## 1. 개요
나는 평소 RPG 게임을 즐기고, 한 명의 플레이어로서 게임을 경험하다 보니, 게임 개발에서는 단순한 개발 능력만큼이나 **유저와의 소통**이 중요하다고 생각하게 됐다.  
이런 관점에서 이번 프로젝트를 기획했다.

---

## 2. 현재 진행도 및 예상 진행 방향

### 2.1 현재 진행도
- 크롤링을 대부분 완료했지만, **부정적인 리뷰**가 중요한데 대부분 긍정적인 리뷰만 존재한다.  
- 따라서 **부정 리뷰가 많은 게임**을 추가로 찾는 중이다.

### 2.2 예상 진행 방향

| 단계 | 목표 | 주요 내용 및 방법론 |
| :--- | :--- | :--- |
| **1. 감성 기반 이분화** | 칭찬 요소와 불만 요소를 명확히 분리 | 긍정/부정 리뷰 그룹별 키워드 추출을 통해 '핵심 매력 요소'와 '시급 보완 요소'를 이분화하여 전략 기초 확보 |
| **2. 민감도 지수** | 유저 분노 요소의 우선순위 결정 | (부정 언급량)이 아닌 **부정 감성 강도**를 활용해 **민감도 지수**를 산출, 객관적인 개선 순위 확정 |
| **3. 복합 연관성 분석** | 불만을 폭발시키는 근본 패턴 규명 | '버그 + 최적화' 등 두 가지 이상 민감 키워드가 동시에 언급된 리뷰를 분석, 복합 문제 원인 파악 |
| **4. 결론** | 리뷰를 크롤링하여 평가 | 분석 결과를 기반으로 게임의 문제점을 파악하고 **시각화** |

## 3. 데이터 확보 및 분석 방법

### 3.1 데이터 수집 및 보완 계획

| 구분 | 내용 |
| :--- | :--- |
| **출처** | Steam, 메타크리틱 |
| **규모** | **약 30,000건 이상**의 이용자 리뷰 크롤링 목표 (부정 리뷰 비율 확보 위해 추가 수집 진행) |
| **특징** | 출시 후 **1개월 이상 경과**한 게임 리뷰 중점 수집 |

### 3.2 핵심 분석 기법 (KoELECTRA 기반)

| 분석 기법 | 상세 내용 | 목적 |
| :--- | :--- | :--- |
| **KoELECTRA 기반 감성 분석** | 한국어 특화 모델로 리뷰 문장별 **부정 감성 강도(확률)** 측정 | 민감도 지수 산출을 위한 핵심 정량 데이터 확보 |
| **키워드 빈도 및 정제** | 단순 명사 추출을 넘어 **개발 관련 전문 용어**, **비속어/은어** 정제 | 유저 언어에서 핵심 불만 키워드 식별 |
| **복합 연관성 지수** | '밸런스'와 '현질' 등 두 개 이상의 키워드 조합에 대한 **분노 증폭 지수** 계산 | 근본적인 유저 불만 패턴 규명 |

## 4. 기대 효과 (프로젝트 결과 예측)

- **Top 5 민감 요소 리스트 도출**  
  데이터 기반으로 **게이머들이 가장 민감하게 반응하는 요소 5가지** 우선순위와 정량적 지표(민감도 지수) 제공  

- **유저 니즈 맵 완성**  
  - 긍정 리뷰 → **유저 만족 핵심 요소**  
  - 부정 리뷰 → **실패 유발 핵심 요소**  
  이를 통해 **실질적인 게임 개발 및 운영 전략 지침** 제공  

- **정책 제언**  
  예: "미흡한 버그 관리에 인력 $X\%$ 우선 투입"  
  **데이터 기반 구체적·실현 가능한 개선 방안** 제시로 프로젝트 실용성 증명
